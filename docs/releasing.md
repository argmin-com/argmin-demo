# Releasing

This repo uses a manual GitHub Actions release workflow plus immutable `v*` tags.

## Prerequisites

- the `Release` workflow is already present on `main`
- the `Protect release tags` ruleset must remain active for `refs/tags/v*`
- `main` must be green before a release is created

## Publish a release

1. Open **GitHub Actions**.
2. Select **Release**.
3. Click **Run workflow**.
4. Enter:
   - `version`: semantic version, for example `0.3.3`
   - `target_ref`: usually `main`
   - `prerelease`: `false` for a full release, `true` for a prerelease

## What the workflow does

- validates the SemVer input
- checks out the requested ref
- runs `ruff`, `mypy --strict`, and `pytest -q`
- builds `sdist` and `wheel` artifacts
- generates release checksums and a manifest
- creates `v<version>` if the tag does not already exist
- publishes the GitHub Release with autogenerated notes

## Release artifacts

Each release publishes:

- `dist/*`
- `requirements.lock`
- `requirements-dev.lock`
- `release-manifest.json`
- `SHA256SUMS.txt`

## Notes

- Release tags are immutable once created.
- If a tag already exists on the wrong commit, the workflow will fail rather than rewrite it.
- If the workflow definition itself changes in the future, that change must be merged to `main` before it can be used.
